# 产品需求 - 稳定固化(不频繁变更)

## 1. 极简摘要

- 产品：Hermes Assistant -- Hermes cron 任务管理平台，展示任务状态与执行历史
- 用户：内部运维人员和开发人员，日常监控和问题排查
- 结构：单页应用，首页展示任务列表 + 统计栏，点击展开执行历史
- 核心流程：启动服务 → 加载任务配置 → 展示任务卡片 → 查看执行详情

产品定位、体验原则与判断准则见 ./01-prd-sense.md。

---

## 2. 页面结构

- 主页面（单页）
    - Header：产品标题 + 手动刷新按钮
    - StatsBar：今日统计汇总（总执行数 / delivered / silent）
    - JobList：任务卡片列表
        - JobCard：任务名称、调度表达式、状态、今日统计
        - RunHistory（展开）：执行时间、响应内容、状态标签
    - Footer：版本信息

---

## 3. 功能需求

### 3.1 任务列表展示
- 从 /api/jobs 获取所有已注册任务
- 每个任务以卡片形式展示：名称、cron 表达式、启用状态、下次执行时间
- 禁用状态的任务灰色半透明显示
- 卡片内展示今日执行统计（total / delivered / silent）

### 3.2 执行历史查看
- 点击任务卡片展开执行历史表格
- 从 /api/runs/{job_id} 获取执行记录（默认最近 20 条）
- 表格列：执行时间、响应内容摘要、状态标签
- 状态标签着色：delivered（绿）、silent（灰）、error（红）
- 响应内容中的 CM 订单号自动生成跳转链接

### 3.3 统计栏
- 页面顶部展示今日所有任务的汇总统计
- 展示指标：总执行次数、delivered 次数、silent 次数
- 数字与对应状态颜色一致

### 3.4 自动刷新
- 页面每 5 分钟自动刷新数据
- 提供手动刷新按钮，点击立即拉取最新数据
- 刷新过程中显示 loading 状态

### 3.5 订单号链接
- 识别响应内容中 CM 开头的订单号（正则 `/CM\d+/g`）
- 生成跳转链接指向 op.zuoyebang.cc 对应订单页

---

## 4. 非功能与约束

- 平台：macOS 桌面浏览器（Chrome/Safari），最小宽度 768px
- 体验：首屏加载 < 2 秒，数据刷新 < 1 秒
- 数据：只读展示 Hermes 输出，不修改任何文件
- 部署：本机运行，launchd 守护进程自动启动
- 安全：仅监听 localhost，无需认证
- 技术规范：JS Airbnb Style Guide、CSS ITCSS 架构 + BEM 命名
